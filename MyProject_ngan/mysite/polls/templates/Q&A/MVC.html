Mô hình MVC

Django hỗ trợ mô hình MVT ( Model – View – Template ), nó cũng tương tự như mô hình MVC giúp quản lý và xây dựng dự án phần mềm có hệ thống hơn.


Chu trình của một request: django nhận được một một request->Dựa vào URL sẽ quyết định view nào sẽ được sử dụng ->View sẽ thao tác với Model lấy dữ liệu từ database sau đó gửi các dữ liệu cần thiết tới Template để Template tạo ra file html và gửi về View sau đó trả về cho người dùng.

Nhưng thực chất mô hình chính xác mà Django triển khai được gọi là MVT (model-view-template). Nó không nhất thiết phải trông như thế nào, nhưng dữ liệu nào được trình bày. Khung nhìn mô tả dữ liệu nào bạn nhìn thấy, không phải cách bạn nhìn thấy dữ liệu đó. Nó khác biệt một cách tinh tế

Ưu điểm:
Dễ dàng cài đặt, cần gì code nấy
Code dễ dọc, do mọi thứ đều thể hiện trong 1 function
Luồng dữ liệu tường minh
Sử dụng decorators rất đơn giản
